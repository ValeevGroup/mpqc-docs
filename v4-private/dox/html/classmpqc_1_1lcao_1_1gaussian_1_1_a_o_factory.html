Detailed Description

template<typename Tile, typename Policy>
class mpqc::lcao::gaussian::AOFactory< Tile, Policy >

Gaussian AO integral factory using ab initio Hamiltonians.

This class computes Gaussian AO integrals exactly and using density fitting

template<typename Tile , typename Policy >
mpqc::lcao::gaussian::AOFactory< Tile, Policy >::AOFactory ( const KeyVal &  kv)

The KeyVal constructor.

Parameters
kvthe KeyVal object, it will be queried for all keywords of the Factory ctor as well as the following additional keywords:
Keyword Type Default Description
screen string dist method of screening; valid values are schwarz (2-norm based), schwarz_inf (infinity-norm based), dist (2-norm based distance-dependent screeners: QQR for 4-center integrals and SQVl for 3-center integrals), dist_inf (same as dist but infinity-norm based), or empty string to turn screening off
screen_threshold real value of Factory's precision screening threshold for integrals (integral tiles with per-element norm smaller than this value will be neglected)
engine_precision real value of Factory's precision the default AO engine precision
inv_sqrt_method string cholesky_then_canonical can take the following values: canonical (Lwdin canonical orthogonalization, with the 1e8 target condition number threshold), cholesky_then_canonical (try the unpivoted Cholesky decomposition via POTRF LAPACK or its ScaLAPACK analog; if that fails revert to canonical), and iterative (iterative computation of inverse square root). canonical[1e7]", "iterative"}. The target condition number for the canonical orthogonalizer can be specified in the square brackets, such as `cholesky_then_canonical[1e7]`. N.B. Many LAPACK implementations do not exit gracefully when matrices are near singular, hence the use of `cholesky_then_canonical` may cause unrecoverable errors; the use of `canonical` is recommended in such scenarios.
f12_factor string see notes If aux_basis exists in BasisRegistry, used to define the F12 correlation factor. Valid formats include:
  • ttg[A] where A is a nonnegative real parameter defining the Ten-no-Type Geminal (TTG): \( - \exp(- A r_{12})/A \).
  • ttg-Ng[A] where N and A are nonzero integer and nonnegative real parameters defining the Ten-no-Type Geminal (TTG) approximated by by fitting to N Gaussian geminals: \( - \exp(- A r_{12})/A \approx \sum\limits_{i=1}^N c_i \exp(- \alpha_i r_{12}^2) \) (see mpqc::lcao::f12::ttg_ng_fit for the detail of the fit).
    If not given, will use TTG with recommended exponent obtained by calling mpqc::lcao::f12::basis_to_f12exponent with the name of VBS (if given; see vir_basis keyword of BasisRegistry) or OBS (see basis keyword of BasisRegistry). In absence of a recommended exponent for the particular OBS/VBS TTG[1.2] is used.
Warning
These keywords should be prefixed by "wfn_world:" (not recommended) or defined directly in this KeyVal object
