KeyVal constructor for SD

Parameters
kvthe KeyVal object; it will be queried for all keywords of AOWavefunction as well as the following additional keywords:
Keyword Type Default Description
charge int 0 the total charge
multiplicity int see Description the spin multiplicity (i.e., \( 2 M_S + 1 \) , where \( M_S \) is the z-axis projection of the spin angular momentum vector); the default is 2, if the value of charge results in an odd number of electrons, else 1; note that multiplicity<=1 (i.e. M_S<0) is supported
kramers_restricted bool true whether to restrict spin-up and spin-down sp states to be conjugates of each other
max_iter int 30 maximum number of iterations
rdm mpqc::RDM::Provider sCI the class to use for obtaining the 1- and 2-RDMs; no need to provide wfn_world for this since this object will provide it; the default is to use a default sCI object
ncore int 0 specifies the number of fully-occupied sp states uncorrelated in the RDM provider (inactive, in MCSCF language)
sd bool false if true, compute single-determinant state
guess_orbitals CanonicalSPSpace<Basis>::Provider see Description provider of trial orbitals by Gaussian-orbitals methods; the default is not provided
real_guess string "NSOAD" if guess_orbitals are not provided, this specifies the guess; valid values are "SAP" (SAP) and "NSOAD" (Fock build on SOAD natural orbitalsin minimal basis)
active array<int> [ncore, ncore+1, ... ] array of (zero-based) indices of guess sp states to select as active sp states
save_basis bool false whether to checkpoint basis
read_basis bool false whether to try reading the basis from checkpoint
plot_basis bool false if true, will plot the basis in the Gaussian Cube format as well as save the functions' tree structure in an ad hoc format
max_degeneracy int 1 if have more than 1 atom this can be used to inform the solver on the max degeneracy of SP states; valid values are between 1 and 5
degeneracy_thresh real 1e-6 the numerical threshold for detecting near-degeneracies in mpqc::math::numeric_range_degeneracies. used in relative comparison
eps_start real wfn_world:eps if given, will dynamically ramp up precision from eps_start to the target, i.e. wfn_world:eps
subspace_maxsize int 4 max size of the subspace kept by KAIN extrapolation
localize bool false whether to localize orbitals or not. only implemented for SD
level_shift double -0.15 the minimum allowed BSH parameter / effective NO energy; any NO with a greater energy is given an effective potential to push the effective energy to the level shift
rampup string "production" if eps_start!=wfn_world:eps, this specifies how the precision is ramped up; valid values are "reference" (tight convergence for each intermediate precision, reset extrapolation after each precision change) and "production" (rampup precision aggressively)
maxrotn real 0.25 Maximum sine of the orbital rotation angle that is deemed acceptable for Lippmann-Schwinger update (if exceeded, will step restrict and/or reset extrapolation) and naturalization (if exceeded, will reset extrapolation)
obs_block_size int 8 the target orbital space block size; use to control parallelism in tensor algebra
xc:functional string none the KS DFT XC functional to use; only makes sense if sd is true; valid values are LDA, and, if configured with ENABLE_LIBXC and libxc found, BP86, PBE, PBE0, and B3LYP
xc:rhotol real 1e-7 electron density values less than this are replaced by the value of xc:rhotol; density munging using this parameter is used to control accuracy in KS DFT
xc:rhomin real 1e-12 electron density values less than xc:rhotol are replaced by the value of this; density munging using this parameter is used to control accuracy in KS DFT
xc:ggatol real 1e-4 where electron density values less than this the gradients of electron density values are replaced by xc:rhotol; density munging using this parameter is used to control accuracy in KS DFT
xc_deriv_type string "abgv" if GGAs are used, this specifies the derivative type; valid values are "abgv" (Alpert-Beylkin-Gines-Vozovoi), "bspline", and "ble" (band-limited exponential)
seed double 1 if using complex solver, this specifies the seed for the std::mt19937 random number generator (WARNING: standard random number generators are not portable); a seed of 0 uses the value returned by std::time() as the seed
debug_size int 0 if > 0, will produce debugging output until determinant set exceeds this size
low_memory bool true reduce code's memory use at the expense of increasing execution time
