Detailed Description

This provides syntactic sugar for Acyclic Visitor used to implement double dispatch pattern.

Note
The need for double dispatch arises when an abstract model (e.g. wave function) needs to compute abstract properties. Instead of polluting the base model class (Wavefunction) by methods for computing every known property under the sun, we implement double dispatch using the Acyclic Visitor pattern. The property classes will be the visitees in such a pattern.
Warning
Note that we are not using the traditional names since they only confuse users. Translation: provider = Visitor, property = Visitable
