KeyVal constructor for Basis::Factory.

Parameters
kvthe KeyVal object will be queried for the following keywords:
Keyword Type Default Description
name string none Specifies the name of the basis set to be used for all atoms. The basis sets are provided by the built-in bundle of the Libint library, or found in the supplemental locations (see description of environment variables LIBINT_DATA_PATH and MPQC_BASIS_PATH below).
element_names array<array<string,string>> or array<array<integer,string>> none Sequence of {element,name} pairs specifies the name of the basis set on per-element basis, with the element specified by its standard (case-insensitive) symbol or its atomic number. For the elements not specified by element_names the name given by name will be used.
names array<string> none Specifies the name of the basis set for every atom. This keyword overrides name and element_names.
bse_basis keyword group none Specifies basis sets using the BSE (Basis Set Exchange) schema format. The KeyVal should contain an elements object with atomic numbers (as strings) as keys, each containing an electron_shells array following the BSE schema. This keyword overrides name, element_names, and names. See the BSEShell class for details on the shell specification format.
cart bool see description if set to false will make all basis functions to be solid harmonics; if set to true will make all basis functions to be Cartesian; if not provided will use the default values (Libint or BSE) for the given basis set
uncontract bool false if set to true will uncontract all AOs
unit_normalize bool true if set to true will rescale all basis functions to be unit normalized (this is the Libint default as well), else use the basis functions as defined in the basis definition
Basis library and custom bases
Basis::Factory uses the library of basis sets bundled with Libint2; see Libint2's basis set library for the available basis sets. The default library location can be overridden by specifying the environment variable LIBINT_DATA_PATH. Basis sets not in that library can be made available as follows:
  • intrusively: by adding basis set files to the Libint2's basis library directory directly. The basis set files must be in the so-called Gaussian format; Basis Set Exchange can be used to obtain many popular basis sets in such format (choose "Gaussian" format, not "Gaussian, system library", and save definition to a file named <lowercase name of the basis>.g94, e.g. basis BaSiS-2023 to be saved to file named basis-2023.g94).
  • non-intrusively: by adding Gaussian-formatted basis set files to the directories given by environment variable MPQC_BASIS_PATH; set MPQC_BASIS_PATH to dir1:dir2:... to query directories dir, dir2, .. (in that order of preference) for basis sets. If a basis is not found in any directory listed in MPQC_BASIS_PATH, the Libint2 library will be queried for it last.
