Detailed Description

template<typename Tile, typename Policy>
class mpqc::lcao::SD< Tile, Policy >

General Single-Determinant Wavefunction.

Supports spin-restricted (including high-spin open-shell) and spin-unrestricted determinants.

KeyVal constructor for SD

Parameters
kvthe KeyVal object; it will be queried for all keywords of AOWavefunction as well as the following additional keywords:
Keyword Type Default Description
charge int 0 the total charge
multiplicity int 2 for odd # of electrons, else 1 the spin multiplicity (i.e., \( 2 S + 1 \) ; SD only supports high-spin states, with all unpaired electrons with spin up (but all-spin-down also supported by setting multiplicity to \( 1-2S \) )
spin_restricted bool true whether to restrict spin-up and spin-down orbitals to be identical
max_iter int 30 maximum number of iterations
fock FockBuilder DirectFockBuilder the FockBuilder object used to construct the Fock matrix; derived classes typically hardwire this object
density_builder string eigen_solve type of DensityBuilder, valid values are eigen_solve (use ESolveDensityBuilder) and purification (use PurificationDensityBuilder)
localizer class none will use this object to localize the occupied orbitals; if given an empty object (or, alternatively, keyword localize is set to true), will use FBLocalizer with default parameters
localize_core bool true if false, core orbitals will not be localized.
clustered_coeffs bool false if true, will recluster coeffs in DensityBuilder; will error if true and localization_method is "bf(valence)" or "rrqr(valence)"
t_cut_c real 0.0 threshold in DensityBuilder, SparsePolicy only
target_fock_precision real std::numeric_limits<double>::epsilon() target precision for FockBuilder
orthog string default if density_builder is set to eigen_solve, this specifies how basis orthonormalization is performed; valid values are cholesky_inverse, inverse_sqrt (iterative sparse matrix multiply), symmetric (symmetric orthogonalization), canonical (canonical orthogonalization), and default (try cholesky_inverse, then try canonical)
target_condition_number real 1e8 if density_builder is set to eigen_solve, this specifies the target condition number of the basis, valid when orthog is set to default (when cholesky fails), symmetric or canonical
_deg_tol real 1e-6 relative comparison threshold for detecting degenerate orbital sets; may need to loosen if the Hamiltonian is particularly noisy
guess string hcore, if using model Hamiltonian, else soad how to prepare guess density; there are 5 valid values: soad or soad_unc (hypnotizingly awesome Superposition Of Atomic Density method; use soad_unc if OBS is densely contracted the guess is constructed with the uncontracted OBS, then recontracted), sap (guess constructed with Superposition of Atomic Potentials; see J. Chem. Phys. 152, 144105 (2020); doi: 10.1063/5.0004046), hcore (core Hamiltonian), and read (read density/coefficients from the checkpoint file specified by chkpt_file)
chkpt bool false whether to save density/coefficients to the checkpoint file given by chkpt_file
chkpt_file string <basename>.sd.chkpt, where <basename> is the value returned by FormIO::default_basename() the name of the file for checkpointing the density/coefficients

Density matrix in Pauli representation

  • D_[0] = spin-average (idempotent) D, i.e. (D(alpha) + D(beta))/2
  • (if needed) D_[1] = spin-difference/"magnetic" density, i.e. (D(alpha) - D(beta))/2
